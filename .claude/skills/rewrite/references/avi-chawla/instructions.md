# Avi Chawla (Daily Dose of DS) 风格 — 重写操作指令

## 何时读取本文件

当用户确认使用 Avi Chawla（Daily Dose of DS）风格重写时，在分析原文之后、开始重写之前读取本文件。

如需更深入了解风格细节，再读取 `compose.md`。如需参考原文进行对比学习，再读取 `example.md`。

---

## 核心效果

**让文章有钩子、有节奏、有数据、不废话。**

## 六段式叙事弧线

根据原文内容，套用以下结构：

| 段落 | 目的 | 长度 |
|------|------|------|
| 钩子 | 反直觉论点，打破预期 | 1-2 句 |
| 数据 | 具体数字支撑 | 1-2 句 |
| 反转 | 否定显而易见的解释 | 2-3 句 |
| 问题 | 铺陈具体痛点场景 | 3-5 组"总-例-果" |
| 悖论 | 指出更深层的矛盾 | 2-3 句 |
| 方案 | 对比数据验证新方案 | 2-4 句 |

**关键约束**（违反任何一条必须重新调整）：
- 每段不超过 2 句话（bullet list 除外）
- **硬性要求**：前 3 个段落必须出现至少一个具体数字或量化指标。如果原文前3段没有数据，必须从后文提取或构造一个可量化的对比放到开头
- **关键数据和结论必须加粗**：用 `**文字**` 标记所有关键数字、核心结论、方案名称。不要全文乱标，只标真正重要的
- 删除所有过渡语：Furthermore, Moreover, In addition, As a result, Therefore, Consequently, 此外, 因此, 综上所述
- 保留原文中的所有图片链接 `![](url)` 和链接 `[text](url)`

## 词汇转换

将以下词汇模式替换：

| 避免 | 使用 |
|------|------|
| utilize / 利用 | use / 使用 |
| facilitate / 促进 | help / 帮助 |
| demonstrate / 展示 | show / 说明 |
| approximately / 大约 | ~ |
| significantly / 显著地 | 直接给数字 |
| 因为...所以... | 直接因果句，或拆成两句 |
| 不仅...而且... | 并列短句 |

## 中文特有适配

如果原文是中文，额外应用：

1. **破折号分隔**：用「——」替代部分换行分隔
2. **直角引号**：关键术语用「」而非 ""  
3. **语气词**：适度加入"老实说"、"实际上"、"说白了"  
4. **段落弹性**：中文允许 1-2 句/段，比英文稍长  
5. **数字保持**：全部使用阿拉伯数字，更显眼  

## 风格自检清单

重写完成后，逐条确认：

- [ ] 开头是否有反直觉论点？
- [ ] **前3段内是否出现具体数据（至少一个数字）？**
- [ ] **关键数据、核心结论、方案名称是否用 `**粗体**` 标记？**
- [ ] 是否有原因反转（否定显而易见的解释）？
- [ ] 段落平均长度是否<=2句话？
- [ ] 是否删除了所有过渡语？
- [ ] 是否有具体场景/例子说明问题？
- [ ] 是否用对比数据（而非形容词）证明观点？
- [ ] 是否保留了原文的所有图片和链接？
- [ ] （中文）语气是否直接、无废话？

如果任何一条未通过，重新调整对应段落，直到全部通过。
