# Distill And Archive

## Overview

将网络文章或内容提炼为精炼知识点，并归档到用户的笔记系统中。工作流程包含：内容提取与 distill、调用 archive 技能执行归档。

**依赖关系**：本技能依赖以下技能：
- **distill-takeaways**：用于提取和整理内容
- **archive**：用于调研笔记系统、制定归档计划、执行归档和提交

## Workflow

### Step 1: Distill 内容

当用户提供 URL 时：

1. 使用 web-reader 工具获取文章内容
2. 进行中英网络深度搜索，补充背景信息和拓展资源
3. 使用 distill-takeaways 技能将内容整理为精炼的知识点（takeaways）

**知识组织原则：**
- **简单知识**（围绕单一核心观点）：保持为一个知识点，不强行拆分
- **复杂知识**（多个独立论点或视角）：拆分为若干关联知识点
- 判断标准：知识点之间是否必须相互引用才能理解？如果是，则属于同一知识单元

**知识与观点的处理：**
- **知识（Knowledge）**：事实性、客观、可验证的内容
- **观点（Opinion）**：作者的主观经验、实践偏好、流派主张
- **处理方式**：采用混合归档，统一使用 `####` 标题格式，通过措辞体现区别：
  - 知识：直接陈述事实（"XX 是..."、"XX 的工作原理是..."）
  - 观点：使用体现主观性的措辞（"XX 建议..."、"XX 的实践是..."、"一种观点认为..."）

**输出格式：**
```markdown
#### [标题]

[内容：使用朴素的单词，精确术语，符合认知习惯，融入轻微情感表达]

见：[链接标题](链接地址)
```

### Step 2: 调用 Archive 技能

Distill 完成后，调用 **archive** 技能执行后续步骤：

1. 将 distill 后的内容传递给 archive 技能
2. archive 技能将执行：
   - 调研笔记系统结构
   - 制定归档计划
   - 等待用户确认
   - 执行归档
   - 提交更改

**调用方式**：
```
/archive [distill 后的内容]
```

或直接在对话中说明：
"现在使用 archive 技能将这些内容归档到笔记系统。"

## 格式规范

沿用 archive 技能的格式规范。

## 注意事项

- **质量优于速度**：Distill 时充分搜索，确保信息准确
- **分工明确**：本技能专注于内容提取和整理，归档操作交给 archive 技能
- **原子提交**：一次归档一个主题，便于追溯
