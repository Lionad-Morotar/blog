# Step 2: 发现互相矛盾内容

## 目标

帮助用户识别多份资料之间的不一致性，判断哪个来源更可信，或理解分歧的根本原因。

## 何时执行

**必须执行的情况：**
- 用户提供 2 份或更多关于同一主题的资料
- 用户问"这些说法有什么矛盾吗"
- 资料明显观点对立（如一篇唱衰一篇看好）

**跳过条件：**
- 只有单一来源
- 用户明确说"这些资料互补，没有矛盾"

## 执行流程

### 1. 建立对比维度

找出资料中都涉及的共同话题，常见维度包括：
- **事实层面**：数据、日期、事件经过
- **因果解释**：为什么会发生
- **预测判断**：未来会怎样
- **价值评估**：好/坏、成功/失败
- **解决方案**：应该怎么做

### 2. 逐点比对

对每一个共同话题，列出各资料的观点：

| 维度 | 资料 A 观点 | 资料 B 观点 | 一致性 |
|-----|------------|------------|--------|
| ... | ... | ... | 一致/矛盾/补充 |

### 3. 矛盾类型判定

识别矛盾的性质：

**A. 事实性矛盾（最严重）**
- 数据不一致（市场份额 30% vs 45%）
- 时间冲突（2023 年 vs 2024 年）
- 事件归因不同（A 公司收购 vs B 公司收购）

**B. 解释性矛盾**
- 同一现象的不同归因
- 不同的因果链条
- 不同的权重分配

**C. 预测性矛盾**
- 对未来的不同判断
- 时间线分歧（短期 vs 长期）

**D. 价值观矛盾**
- 对同一事件的不同评价
- 利益立场导致的观点差异

### 4. 可信度评估

对每个矛盾点，评估哪方更可信：

| 评估维度 | 资料 A | 资料 B |
|---------|--------|--------|
| 证据强度 | 实验数据 | 专家意见 |
| 利益相关性 | 第三方独立 | 利益相关方 |
| 时效性 | 2024 最新 | 2022 过时 |
| 方法论 | 大样本 | 小样本 |
| 同行评议 | 是 | 否 |

### 5. 分歧解释

如果双方都有一定可信度，分析分歧原因：
- **时间因素**：不同时期的观察结果
- **样本差异**：针对不同人群/地区
- **定义差异**：对关键概念理解不同
- **利益立场**：企业 PR vs 学术独立
- **深度差异**：表面观察 vs 深度调查

## 输出格式

```markdown
## 多源资料矛盾分析报告

### 资料对比矩阵

| 话题 | 资料 A | 资料 B | 资料 C | 关系 |
|-----|-------|-------|-------|------|
| 市场规模 | 1000亿 | 800亿 | - | A vs B 矛盾 |
| 增长驱动 | 技术突破 | 政策支持 | 两者皆有 | 互补 |

### 核心矛盾点

#### 矛盾 1：[简要描述]
- **资料 A 立场**：...
- **资料 B 立场**：...
- **矛盾类型**：事实性/解释性/预测性
- **更可信方**：[A/B/无法判断] + 理由
- **可能解释**：[如果无法判断]

### 综合建议
- 哪些论断可以采信：...
- 哪些需要进一步验证：...
- 可能的整合视角：...
```

## 提示词

**中文：**
```
比较这些资料，找出它们之间所有相互矛盾的地方。对于每一个矛盾之处，解释哪个资料的证据更有力，以及原因。如果两者都可信，解释可能导致分歧的因素。

按以下结构输出：
1. 资料概览（来源、立场、目的）
2. 逐话题对比矩阵
3. 详细矛盾分析（每个矛盾单独一节）
4. 可信度评估与建议
```

**English:**
```
Compare these sources and identify every point where they contradict each other. For each contradiction, explain which source has stronger evidence and why. If both are credible, explain what factors might explain the disagreement.

Structure your output:
1. Source overview (origin, stance, purpose)
2. Topic-by-topic comparison matrix
3. Detailed contradiction analysis (separate section for each)
4. Credibility assessment and recommendations
```

## 常见场景处理

### 场景 A：学术论文 vs 产业报告
- 学术：严谨但可能滞后
- 产业：前沿但可能有 PR 成分
- 处理：学术研究作为基准，产业报告补充最新动态

### 场景 B：竞争对手资料
- 如 Apple vs Samsung 的报告
- 处理：交叉验证，寻找第三方独立数据

### 场景 C：不同时期的资料
- 旧资料可能已过时
- 处理：标注时间，优先采信最新数据
