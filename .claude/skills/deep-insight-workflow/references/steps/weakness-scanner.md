# Step 3: 弱点扫描器

## 目标

帮助用户识别资料中的方法论缺陷、逻辑漏洞和过度断言，评估分析的可靠性。

## 何时执行

**必须执行的情况：**
- 用户需要对资料进行"质量检查"
- 资料将被用于重要决策（投资、政策、战略）
- 资料声称"颠覆性"或"突破性"发现
- 用户问"这篇分析靠谱吗"

**执行重点：**
即使来源可靠（如 Nature 论文），也可能存在方法局限。这一步不是"找茬"，而是帮助用户理解"这份分析的边界在哪里"。

## 执行流程

### 1. 方法论审查

检查研究设计中的潜在问题：

| 审查项 | 检查内容 | 红旗信号 |
|-------|---------|---------|
| 样本代表性 | 样本是否反映总体 | 大学生样本推及全体人群 |
| 对照组设置 | 是否有适当的对照 | 无对照组或对照不当 |
| 变量控制 | 混淆变量是否控制 | 多因素交织却做单因素分析 |
| 测量工具 | 测量方法是否有效 | 未经验证的自定义量表 |
| 时间维度 | 追踪时间是否充分 | 短期观察推长期结论 |
| 数据完整性 | 缺失数据处理 | 大量删失数据未说明 |

### 2. 统计分析审查

检查统计方法的合理性：

- **p-hacking 迹象**：多测试但未校正 α 水平
- **效应量 vs 显著性**：统计显著但实际效应微弱
- **因果关系推断**：相关当因果
- **样本量 adequacy**：小样本却做大推論
- **多重比较问题**：未进行 Bonferroni 等校正

### 3. 逻辑漏洞扫描

识别论证中的逻辑错误：

| 逻辑谬误 | 示例 |
|---------|------|
| 稻草人谬误 | 曲解对方观点再攻击 |
| 滑坡谬误 | A 发生就会导致 Z，忽略中间环节 |
| 虚假两难 | 只有 A 或 B 两种选择 |
| 诉诸权威 | "某专家说"但领域不匹配 |
| 幸存者偏差 | 只看到成功案例 |
| 后此谬误 | 因为 A 在 B 前，所以 A 导致 B |
| 诉诸无知 | "没有证据反对"="证据支持" |

### 4. 过度断言识别

标记超出证据支持的声明：

**断言强度层级：**
- 推测级（may/might）：证据不足但有合理推测
- 趋势级（suggests/indicates）：有一定证据支持
- 结论级（shows/demonstrates）：强证据支持
- 确证级（proves/confirms）：几乎确定

**审查要点：**
- 动词强度是否与证据匹配？
- 限定词是否被不当省略？
- 边界条件是否被忽略？

### 5. 证据缺口分析

找出论证中缺失的关键环节：

```
论证链条：A → B → C → D

检查：
- A→B 是否有证据？
- B→C 是否成立？
- C→D 是否跳过步骤？
- 是否有隐藏的中间假设？
```

## 提示词

**中文：**
```
扮演严苛的同行评审员角色。指出这些资料中所有的方法论缺陷、逻辑漏洞、夸大其词和缺乏依据的推断。针对每一个缺陷，提出需要哪些补充证据来加强论点。

按以下结构输出：
1. 方法论审查清单及发现
2. 逻辑漏洞逐项分析
3. 过度断言识别与修正建议
4. 证据缺口列表
5. 总体可靠性评级（1-5 星）+ 使用建议
```

**English:**
```
Act as a harsh peer reviewer. Identify every methodological flaw, logical gap, overclaim, and unsupported leap in these sources. For each weakness, suggest what additional evidence would be needed to strengthen the argument.

Structure your output:
1. Methodology review checklist with findings
2. Logical flaw analysis (item by item)
3. Overclaim identification and correction suggestions
4. Evidence gap list
5. Overall reliability rating (1-5 stars) + usage recommendations
```

## 执行技巧

### 保持建设性
不要只是批评，要告诉用户"如何验证"或"寻找什么补充信息"。

### 区分严重程度
不是所有问题都同等重要。标记哪些是致命的，哪些是小瑕疵。

### 提供替代解释
如果某个推论有问题，尝试提供其他可能的解释。

### 注意领域差异
医学研究需要 RCT，商业分析可以接受专家判断。不要用同一套标准苛求所有领域。

## 常见陷阱

- **过度批判**：顶级研究也有局限，不要因小瑕疵否定整体价值
- **忽视上下文**：考虑资料的目标读者和目的
- **混淆"不完美"与"不可信"**：没有完美的研究，关键是问题是否影响核心结论
