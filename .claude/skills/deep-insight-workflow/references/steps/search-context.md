# Step 0: 背景信息搜索（可选增强）

## 目标

在深入分析用户提供的资料前，通过系统性搜索获取更广泛的背景信息，建立领域认知基准，为后续分析提供参照系。

## 何时执行

**建议执行的情况：**
- 用户对资料主题的熟悉程度不高
- 资料涉及专业领域，需要了解主流观点
- 需要验证资料中的论断是否具有普遍性
- 需要了解该主题的最新发展和争议

**可以跳过的情况：**
- 用户明确说"直接分析，不需要搜索"
- 资料本身已是全面综述
- 时间紧迫，需要快速筛选

## 执行流程

### 1. 构建搜索查询集群

基于资料主题，构建3-5个搜索角度：

| 角度 | 目的 | 示例查询 |
|------|------|---------|
| **领域概览** | 了解主题背景 | "工业园区高质量发展 政策背景" |
| **主流观点** | 了解行业共识 | "工业园区发展 专家观点 趋势" |
| **争议分歧** | 发现不同立场 | "工业园区发展 问题 挑战 争议" |
| **最新动态** | 了解近期发展 | "2024 2025 工业园区政策 新规" |
| **对比视角** | 国际/横向比较 | "国外产业园区发展经验 对比" |

### 2. 并行搜索执行

使用 Task 工具并行派遣搜索代理：

```yaml
name: search-agent-{n}
subagent_type: general-purpose
prompt: |
  搜索主题：{query}

  背景：正在分析一份关于[主题]的资料，需要了解更广泛的背景信息。

  任务：
  1. 使用 WebSearch 搜索相关信息
  2. 对前3-5个相关结果使用 WebFetch 获取详细内容
  3. 提取关键信息：主流观点、数据支撑、争议点、最新趋势
  4. 评估信息来源可靠性

  输出格式：
  ```
  ## 搜索结果: {query}

  ### 来源摘要
  | 来源 | 类型 | 可靠性 | 关键信息 |
  |------|------|--------|---------|

  ### 主流观点
  - [观点1]: 支撑证据...
  - [观点2]: 支撑证据...

  ### 争议/分歧
  - [争议点]: 不同立场...

  ### 与资料相关的背景
  - [相关信息]...
  ```
```

### 3. 结果整合

将搜索结果整理为背景知识库：

```markdown
## 背景知识库

### 领域主流认知
[该领域专家普遍认同的观点]

### 关键数据/事实
[重要的统计数据、时间线]

### 活跃争议
[尚未达成共识的问题]

### 与待分析资料的关联
[背景信息如何帮助理解资料]
```

## 输出使用方式

搜索背景信息将用于：
1. **verify-sources**: 提供交叉验证的参照
2. **expert-insights**: 对比资料观点与主流观点的差异
3. **find-contradictions**: 发现资料与外部信息的矛盾
4. **construct-framework**: 补充领域标准框架

## 提示词

**中文：**
```
基于以下资料主题，构建3-5个搜索查询，全面了解该领域的背景信息、主流观点和最新动态。

资料主题: [主题]
资料关键内容: [摘要]

搜索角度建议：
1. 领域概览（背景知识）
2. 主流观点（专家共识）
3. 争议分歧（不同立场）
4. 最新动态（近期发展）
5. 对比视角（国际/横向）

输出：
1. 搜索查询列表（每个角度1-2个查询）
2. 搜索优先级（哪些角度最重要）
3. 预期获得的信息类型
```

## 注意事项

- **时间控制**: 背景搜索应控制在5-10分钟内，避免过度搜索
- **来源多样**: 兼顾官方、学术、行业媒体等不同来源
- **关联聚焦**: 始终围绕待分析资料的主题，避免发散
- **记录来源**: 所有搜索结果需记录URL和来源类型，便于追溯

## 与 reAct 循环的结合

背景搜索可以作为 reAct 循环的第一轮：
1. 初始资料分析 → 发现问题/疑问
2. **搜索背景** → 获取相关信息
3. 整合新信息 → 更新分析
4. 如仍有疑问 → 第二轮搜索
