# Step 6: 专家级提炼

## 目标

帮助用户从资料中提取该领域专家会认可的突破性核心洞见，识别真正的价值所在。

## 何时执行

**必须执行的情况：**
- 用户问"这些资料的真正价值是什么"
- 需要从大量信息中提炼精华
- 资料涉及专业领域，需要"内行视角"
- 用户准备向他人汇报核心发现

**核心价值：**
不仅总结"说了什么"，更要指出"为什么重要"和"挑战了什么传统观念"。

## 执行流程

### 1. 领域专家角色设定

根据资料主题，扮演相应领域的资深专家（15 年经验）：

| 资料主题 | 专家角色 | 核心关切 |
|---------|---------|---------|
| AI 技术 | AI 研究员/工程师 | 技术突破、范式转变 |
| 商业战略 | 战略顾问/投资人 | 竞争格局、商业模式 |
| 医学研究 | 临床医生/研究员 | 疗效证据、安全性 |
| 社会趋势 | 社会学家/分析师 | 深层动因、长期影响 |
| 经济政策 | 经济学家/政策研究员 | 机制有效性、副作用 |

### 2. 洞见识别标准

专家会立即认可的洞见通常具备：

**突破性标准：**
- 挑战该领域的主流假设
- 提供全新的解释框架
- 揭示隐藏的因果关系
- 预测被忽视的发展趋势

**可识别性标准：**
- 领域内人士会觉得"明显"但外人常忽视
- 符合专家直觉但需要数据验证
- 能解释过去困惑的现象

### 3. 洞见提炼流程

**步骤 1：去噪**
- 排除常识性内容
- 排除媒体炒作/过度解读
- 排除缺乏实质信息的陈述

**步骤 2：对比**
- 资料观点 vs 领域主流观点
- 找出差异点和突破点

**步骤 3：深层挖掘**
- 追问"那又怎样？"
- 追问"这意味着什么变化？"
- 追问"传统观念错在哪里？"

**步骤 4：验证**
- 这个洞见是否被资料充分支持？
- 专家真的会认可吗？
- 是否有反例或限制条件？

### 4. 洞见结构化

每个核心洞见包含：

| 要素 | 说明 |
|-----|------|
| 洞见陈述 | 一句话核心发现 |
| 为什么重要 | 对领域/实践的意义 |
| 挑战的传统观念 | 颠覆了什么旧认知 |
| 支持证据 | 资料中的关键依据 |
| 限制条件 | 适用边界、例外情况 |

### 5. 优先级排序

最多提炼 3 个核心洞见（认知负荷限制）：

**排序标准：**
1. 颠覆性程度（挑战传统观念的强度）
2. 影响范围（影响的领域广度）
3. 证据强度（资料支持的充分性）
4. 时效性（当下相关度）

## 输出格式

```markdown
## 核心洞见提炼

### 专家视角说明

以 [领域] 资深专家（15 年经验）视角分析，这些资料揭示了以下突破：

---

### 洞见 1：[一句话标题]

**核心陈述**：
[简洁有力的洞见陈述]

**为什么重要**：
- 对领域发展的意义：...
- 对实践/决策的影响：...

**挑战的传统观念**：
过去认为：[旧观点]
现在发现：[新认知]
这种转变意味着：...

**支持证据**（来自资料）：
- 证据 1：...
- 证据 2：...

**限制条件**：
- 适用范围：...
- 不适用场景：...

---

### 洞见 2：[一句话标题]
...

### 洞见 3：[一句话标题]
...

---

### 洞见之间的关联

这三个洞见如何构成整体认知：
- 洞见 1 是前提/基础
- 洞见 2 是延伸/应用
- 洞见 3 是前瞻/预测

### 专家级评价

如果向同领域专家展示这些资料：
- 他们会觉得最有价值的是：...
- 他们可能会质疑的是：...
- 他们会追问的是：...
```

## 提示词

**中文（替换 [领域]）：**
```
您是一位拥有15年经验的[领域]专家。请分析这些资料，并找出该领域从业者一眼就能看出具有突破性意义的3个核心见解。对于每个见解，请解释其重要性以及它挑战了哪些传统观念。

输出要求：
1. 专家角色设定（明确领域和经验背景）
2. 3 个核心洞见（每个包含：陈述、重要性、挑战的传统观念、支持证据、限制条件）
3. 洞见优先级说明（为何选这 3 个）
4. 洞见之间的逻辑关联
5. 专家视角的评价（同行会如何反应）
```

**English（替换 [field]）：**
```
You are a [field] expert with 15 years of experience. Analyze these sources and identify the 3 core insights that practitioners in this field would immediately recognize as groundbreaking. For each insight, explain why it matters and what conventional wisdom it challenges.

Output requirements:
1. Expert role definition (domain and experience)
2. 3 core insights (each with: statement, importance, challenged wisdom, evidence, limitations)
3. Priority justification (why these 3)
4. Logical connections between insights
5. Expert evaluation (how peers would react)
```

## 执行检查清单

- [ ] 是否明确了专家角色？
- [ ] 是否排除了常识性内容？
- [ ] 每个洞见是否挑战了传统观念？
- [ ] 重要性说明是否具体而非空泛？
- [ ] 是否有充分的证据支持？
- [ ] 是否标注了限制条件？
- [ ] 3 个洞见是否互补而非重复？

## 常见陷阱

- **过度解读**：资料没有明确支持的"洞见"
- **不够突破**：提炼的只是普通结论
- **过于狭窄**：只关注技术细节而忽视更大图景
- **忽视反例**：为了洞见而忽略限制条件

## 示例对比

**差示例**：
- "AI 很重要"（太泛，不是洞见）
- "这些资料显示 AI 发展很快"（描述性，非突破性）

**好示例**：
- "大模型的涌现能力表明，量变到质变的阈值比预期更低，这挑战了'必须设计专门架构才能解决复杂问题'的传统认知"
