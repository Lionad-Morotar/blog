# Step 4: 挖掘假设条件

## 目标

帮助用户发现资料中隐含但未被明说的假设，评估这些假设对结论的影响。

## 何时执行

**必须执行的情况：**
- 资料做出预测性论断（"将会..."）
- 资料提出政策建议（"应该..."）
- 用户问"什么情况下这个分析会失效"
- 资料依赖模型或理论框架

**核心价值：**
识别"如果 X 不成立，整个结论就崩塌"的那些隐藏的 X。

## 执行流程

### 1. 假设类型识别

资料中的假设通常分为以下几类：

#### A. 事实假设（关于现状）
- 某个市场目前的状态
- 某种技术的当前水平
- 某群体目前的偏好/行为

#### B. 趋势假设（关于未来）
- 增长率将保持稳定
- 技术将继续按当前轨迹发展
- 政策环境不会剧变

#### C. 因果假设（关于机制）
- A 导致 B 的机制
- 干预措施将产生预期效果
- 某些变量之间没有混淆因素

#### D. 价值假设（关于偏好）
- 什么是"好"的结果
- 哪些因素应该被优先考虑
- 不同利益的权重分配

#### E. 边界假设（关于范围）
- 结论适用于什么场景
- 对什么人群/地区有效
- 时间范围是多久

### 2. 挖掘技术

使用以下问题挖掘隐藏假设：

**逆向追问：**
- "如果结论是 X，那必须什么为真？"
- "如果结论错误，最可能是哪里出问题？"

**边界测试：**
- "在什么情况下这个论断不成立？"
- "反例是否存在？"

**替代假设：**
- "还有什么其他解释？"
- "如果采用不同理论框架，结论会变吗？"

**极端情况：**
- "如果关键参数翻倍/减半？"
- "如果关键条件完全消失？"

### 3. 假设评估矩阵

对每个识别出的假设，评估：

| 假设 | 类型 | 重要性(1-10) | 可能错误概率 | 如果错误的影响 |
|-----|-----|------------|------------|--------------|
| ... | ... | ... | 高/中/低 | ... |

**重要性评估标准：**
- 10：假设不成立则核心结论完全崩塌
- 7-9：假设不成立会显著削弱结论
- 4-6：影响次要结论或具体建议
- 1-3：影响很小，可忽略

**错误概率评估：**
考虑历史数据、专家共识、不确定性程度。

### 4. 敏感性分析

识别"单点故障"假设——如果它们出问题，整个分析都会失效。

```
核心结论 ← 依赖 ← 关键假设
              ↓
         如果该假设错误：
         - 结论变为：...
         - 应采取的行动变为：...
```

## 输出格式

```markdown
## 假设挖掘报告

### 事实假设

| 假设 | 资料中的依据 | 重要性 | 可靠性评估 | 失效后果 |
|-----|------------|-------|-----------|---------|
| 当前市场容量为 X | 引用某报告 | 8 | 中等（报告已过时）| 规模预测全部错误 |

### 趋势假设

| 假设 | 隐含前提 | 重要性 | 风险因素 |
|-----|---------|-------|---------|
| 技术将继续指数增长 | 无物理/经济瓶颈 | 9 | 能源限制、材料稀缺 |

### 因果假设

| 假设 | 机制说明 | 替代解释 |
|-----|---------|---------|
| A 导致 B | ... | C 可能是真正原因 |

### 价值假设

| 假设 | 体现位置 | 不同价值观下的分歧 |
|-----|---------|------------------|
| 效率优先于公平 | 第 3 章建议 | 如重视公平，建议应相反 |

### 边界假设

| 假设 | 隐含范围 | 超出范围的风险 |
|-----|---------|--------------|
| 适用于发达国家 | 未提及发展中国家 | 直接套用可能失败 |

### 高风险假设总结

**单点故障假设**（重要性≥8 且 错误概率≥中）：
1. ... - 如果错误，结论变为...
2. ... - 建议增加对冲策略

### 假设验证建议

需要用户确认或额外研究的假设：
1. ... - 建议验证方式：...
2. ...
```

## 提示词

**中文：**
```
找出这些资料中所有未明确说明的假设。对于每个假设，评估其重要性(1-10)以及其错误的可能性。解释如果该假设为假，将会发生什么变化。

按以下结构输出：
1. 按类型分类的假设清单（事实/趋势/因果/价值/边界）
2. 每个假设的重要性评分和理由
3. 错误概率评估及依据
4. 敏感性分析（哪些假设是"单点故障"）
5. 假设失效的替代情景
6. 验证建议（如何验证关键假设）
```

**English:**
```
Identify every unstated assumption in these sources. For each assumption, rate how critical it is (1-10) and how likely it is to be wrong. Explain what would change if that assumption were false.

Structure your output:
1. Categorized assumption list (factual/trend/causal/value/boundary)
2. Importance rating (1-10) with justification for each
3. Error probability assessment with rationale
4. Sensitivity analysis (which are "single points of failure")
5. Alternative scenarios if assumptions fail
6. Validation recommendations (how to verify key assumptions)
```

## 执行检查清单

- [ ] 是否从结论倒推了必要假设？
- [ ] 是否考虑了不同类型的假设？
- [ ] 重要性评分是否有明确依据？
- [ ] 是否识别了"单点故障"假设？
- [ ] 是否提供了假设失效的替代情景？
- [ ] 是否给出了验证建议？

## 常见场景

### 商业预测报告
常见假设：市场将继续增长、竞争格局不变、技术如期落地

### 政策建议
常见假设：政策执行无障碍、目标群体行为可预测、副作用可控

### 学术论文
常见假设：样本代表性、测量工具有效性、理论框架适用性

## 注意事项

- **不要过度解读**：有些假设是合理的默认前提，不需要一一质疑
- **区分显性/隐性**：只关注真正"隐藏"的假设，明说的不算
- **保持平衡**：既指出风险，也承认合理假设的价值
