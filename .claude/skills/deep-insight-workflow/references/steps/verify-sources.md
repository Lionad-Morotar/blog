# Step 1: 核实信息来源

## 目标

帮助用户验证资料质量，识别高风险论断。在深入分析前，先判断资料是否值得投入时间。

## 何时执行

**必须执行的情况：**
- 用户提供来源不明的资料（社交媒体帖子、自媒体文章、匿名博客）
- 资料包含颠覆性论断或惊人数据
- 用户准备基于这些资料做重要决策

**可以跳过的情况：**
- 资料来自顶级期刊（Nature/Science/Cell 等）
- 官方技术文档或 API 参考
- 用户明确说"来源可靠，直接分析"

## 执行流程

### 1. 提取主要论断

从资料中识别所有关键声明，特别是：
- 统计数据和量化结论
- 因果关系的断言（X 导致 Y）
- 预测性论断（未来将...）
- 与常识相悖的观点

### 2. 评估证据强度

对每个论断，追溯其支持证据的类型：

| 证据类型 | 强度等级 | 说明 |
|---------|---------|------|
| 荟萃分析 (Meta-analysis) | ★★★★★ | 多项研究的系统综述 |
| 实验证据 (Experimental) | ★★★★☆ | 对照实验、随机试验 |
| 相关性证据 (Correlational) | ★★★☆☆ | 统计关联但非因果 |
| 轶事证据 (Anecdotal) | ★★☆☆☆ | 个人经历、个案 |
| 无证据/引用缺失 | ★☆☆☆☆ | 纯断言 |

### 3. 外部搜索验证

对于关键论断，启动外部搜索进行交叉验证：

**验证策略：**
| 论断类型 | 搜索策略 | 预期验证结果 |
|---------|---------|-------------|
| 统计数据 | 搜索原始数据源 | 确认数据出处和统计口径 |
| 政策文件 | 搜索官方发布 | 确认文件真实性和完整内容 |
| 专家观点 | 搜索专家其他发言 | 确认观点一致性和专家资质 |
| 趋势预测 | 搜索多方观点 | 了解是否为行业共识或一家之言 |

**搜索代理配置：**
```yaml
name: verify-search-agent
subagent_type: general-purpose
prompt: |
  验证以下论断的外部信息：
  论断: "{claim}"
  来源: "{source}"

  任务：
  1. 使用 WebSearch 搜索该论断的相关信息
  2. 重点关注：官方来源、权威媒体、学术论文
  3. 评估该论断在外部信息中的呈现方式
  4. 标记任何矛盾或不一致之处

  输出格式：
  ```
  ## 验证结果: {claim}

  ### 外部来源
  | 来源 | 一致性 | 备注 |
  |------|--------|------|

  ### 验证结论
  - 可验证性: [高/中/低]
  - 一致性: [一致/部分一致/矛盾]
  - 置信度调整: [原置信度] → [调整后]

  ### 风险提示
  - [如有发现]
  ```
```

**并行验证：** 对3-5个关键论断同时启动搜索验证代理。

### 4. 标记风险点

标记以下危险信号：
- **高置信度 + 弱证据**：用"我确信..."表达，但只有轶事支持
- **循环引用**：资料 A 引用资料 B，B 又引用 A
- **权威诉求**："专家说"但不指明是谁
- **时效性问题**：引用过时数据支持当下论断
- **外部验证失败**：搜索结果与资料论断不一致（新增）
- **信息孤岛**：搜索结果几乎找不到相关提及（新增）

### 5. 输出结构化报告

向用户呈现：
```
## 信息来源核实报告

### 资料概览
- 来源类型：[期刊/博客/视频/书籍等]
- 作者背景：[相关领域资质]
- 发布日期：[时效性评估]

### 论断核查表

| 论断 | 证据类型 | 置信度 | 风险标记 |
|-----|---------|--------|---------|
| ... | ... | 高/中/低 | 如有问题 |

### 建议
- 是否值得深入分析：[是/否/有条件]
- 需要补充验证的点：[具体说明]
```

## 提示词

**中文：**
```
针对这些资料中的每一项主要论断，提取其支持证据并评估其强度（轶事证据、相关性证据、实验证据、荟萃分析证据）。标记出任何证据薄弱但置信度很高的论断。

按以下格式输出：
1. 资料来源信息（作者、发布平台、日期）
2. 主要论断清单（3-7个核心观点）
3. 每个论断的证据评估
4. 风险标记总结
```

**English:**
```
For every major claim in these sources, extract the supporting evidence and rate its strength (anecdotal, correlational, experimental, meta-analysis). Flag any claims with weak evidence that are stated with high confidence.

Output in this format:
1. Source information (author, platform, date)
2. List of major claims (3-7 core assertions)
3. Evidence assessment for each claim
4. Risk flag summary
```

## 示例场景

**用户输入：** "这篇公众号文章说 AI 将在 3 年内取代所有程序员"

**LLM 执行：**
1. 识别主要论断："3 年内"、"取代"、"所有程序员"
2. 评估证据：文章引用的是某创业公司 CEO 的预测，无实证数据
3. 标记风险：高置信度表述（"将"）+ 轶事证据（个人预测）
4. 向用户报告："该论断基于个人预测，缺乏数据支撑，建议谨慎对待或寻找更权威来源"

## 常见陷阱

- **不要为了批判而批判**：顶级期刊论文也有局限性，不要过度质疑
- **注意领域差异**：医学研究需要 RCT，但技术趋势分析可以依赖专家判断
- **区分事实与观点**："数据显示 X"vs"我认为 X"，后者不需要同等证据强度
- **警惕"白大褂效应"**：资料带引用链接≠可信，需点击验证链接实际内容（MIT 研究警示）
- **流畅性偏见**：写得通俗易懂的内容不一定权威，优先采用官方文档/学术来源
- **验证懒惰假设**：假设读者不会点击链接，关键主张需独立验证而非依赖引用格式
