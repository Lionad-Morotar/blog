Governance

Agent Governance

智能体（Agent）相关的政策法规、治理框架与合规要求

Definition

智能体的官方定义

国家网信办、国家发改委、工信部于 2026 年 5 月联合印发的《智能体规范应用与创新发展实施意见》将智能体定义为：

具备自主感知、记忆、决策、交互与执行能力的智能系统，是人工智能产品及服务的重要形态。

该定义强调了智能体的五大核心能力维度（感知、记忆、决策、交互、执行），并将其定位为 AI 产品服务的"重要形态"，反映了大模型时代智能体从概念走向主流的政策认可。

见：财联社政策解读

Principles

智能体发展的四大基本原则

《实施意见》提出四项基本原则，构成政策的核心价值框架：

  1. 安全可控 — 将智能体安全、可靠、可信作为发展的底线要求，贯穿技术研发、应用部署与推广的全过程
  2. 规范有序 — 适应技术演进规律，构建与现有政策法规衔接顺畅、底线红线清晰的治理体系
  3. 创新驱动 — 加强理论创新、技术创新、工程创新联动，体系化突破智能体关键技术
  4. 应用牵引 — 围绕科学研究、产业发展、提振消费、民生福祉、社会治理等实际需求，先易后难、循序渐进

四项原则的排序具有信号意义：安全可控置于首位，表明在"人工智能+"行动中，风险防范优先于技术应用。

Foundation

技术底座与标准协议

《实施意见》从两个维度夯实智能体发展基础：

完善技术底座：

  • 持续提升通用基础模型性能，支持细分领域专用模型，形成模型产品矩阵
  • 提升高质量数据集供给能力
  • 加强任务理解、任务规划、工具使用、长期记忆、互认互通、群体协同等技术攻关
  • 完善智能体研发、测试、部署、运维等工具链

构建标准协议：

  • 制定智能体标准化工作指导文件，系统布局关键技术、数据交换、应用场景、质量评测、安全保障等标准体系
  • 推广智能体互联协议（AIP）等国家标准
  • 布局发展智能互联网：探索建立智能体注册平台，提供数字身份管理、检索发现、能力声明等服务
  • 研究智能体身份标识、可信互联、合规支付、安全防护、冲突解决等基础技术

智能互联网的前瞻布局

《实施意见》提出"布局发展智能互联网"，这是政策中最具前瞻性的概念之一：

  • 建立智能体注册平台，提供智能体数字身份管理、检索发现、能力声明等服务
  • 提升多智能体协同能力
  • 发挥 IPv6 技术优势，提升智能体端到端通信能力
  • 探索建立智能互联网监测指标体系

这一布局意味着智能体将从"孤立应用"走向"互联生态"，类似于移动互联网时代的应用商店模式，但承载的是具备自主决策能力的智能体。

Safety

安全治理框架

《实施意见》构建了四维安全治理体系：

明确产品准则：

  • 完善政策法规和伦理规范，防止智能体利用数据优势、人格化技术实施传播不良价值观、算法压榨等行为
  • 厘清用户本人决策、用户授权决策和智能体自主决策的合理边界
  • 发展规则内嵌、行为围栏等技术，确保智能体在不同场所的行为合法合规

防范安全风险：

  • 提升内生安全能力：数据安全、个人信息保护、攻击检测、权限管理等
  • 加强供应链安全：制定全周期安全规范，建立供应链安全信息共享和预警机制
  • 化解应用衍生风险：完善常态化风险识别、预警及干预机制，防范自动化攻击、隐私侵犯、虚假信息传播等

完善治理体系：

  • 构建分类分级治理框架：敏感领域实行备案、检测、召回管理；低风险领域通过合规自测、信息报告、平台管理实现高效治理
  • 健全合规服务体系：开展第三方评测，推动认证与检测结果互通互认

强化行业自律：

  • 鼓励行业组织、主要企业联合制定自律规则
  • 探索信用评价机制，对技术滥用、诱导消费、虚假宣传等行为进行信用评价

Applications

19 个典型应用场景

《实施意见》围绕五大方向提出 19 个典型应用场景：

方向场景
科学研究科研探索、研发辅助（含软件开发智能体）
产业发展智能制造、能源资源、交通运输、农业生产、金融服务
提振消费终端应用（手机/电脑/汽车/家居/可穿戴/机器人）、文化旅游、商业服务
民生福祉教育教学、医疗健康、人力资源、信息服务
社会治理政务服务、司法服务、公共安全、城市治理、招标投标

关键信号：

  • 软件开发智能体被明确列为研发辅助场景，标志着编程 Agent 获得政策背书
  • 具身智能体在消费服务（家政、养老、托育）和公共安全（灾害救援、安防巡检）领域被单独提及
  • 政务服务从"人找服务"向"服务找人"转型的表述，体现了主动式智能体的政策预期

Ecosystem

创新生态建设

《实施意见》从产业合作和应用推广两个维度部署生态建设：

促进产业合作：

  • 培育开源创新力量：引导开源社区加强智能体布局，开展与开源芯片、操作系统、大模型的兼容适配
  • 搭建产业协作平台：发挥生态联盟、技术验证实验室作用，开展共性技术研发、标准制定、评估认证

强化应用推广：

  • 构建应用推广渠道：推动建立智能体软件商店、行业供需信息发布平台
  • 推进重点场景开放：在产业集聚区、重点行业开展应用试点，打造示范项目
  • 积极培育全球生态：依托世界人工智能大会等国际平台，推动终端设备、软件企业适配智能体，引导企业做好海外合规建设

Policy Context

政策背景与目标

《实施意见》是落实国务院《关于深入实施"人工智能+"行动的意见》的具体举措。国务院文件提出到 2027 年，新一代智能终端、智能体等应用普及率超 70% 的阶段性目标。

答记者问中明确，《实施意见》旨在统筹发展与安全——既促进智能体规范应用和创新发展，又应对智能体高自主性、高权限带来的隐私泄露、越权操作、行为失控等安全风险。
